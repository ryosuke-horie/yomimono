# Linear連携ルール

このプロジェクトではLinearを使用してプロジェクト管理を行っています。

## プロジェクト情報

- **ワークスペース**: edge-work
- **プロジェクト**: 読み物保守
- **ラベル**: 読み物

## ブランチ命名規則

作業を開始する際は、LinearのイシューIDをブランチ名として使用します。

```bash
# 例: EDG-289というイシューに対応する場合
git checkout -b EDG-289
```

**ルール**:
- ブランチ名はLinearのイシューID（例: `EDG-289`）をそのまま使用する
- `feature/`や`fix/`などのプレフィックスは付けない
- イシューIDのみでブランチを作成することで、Linearとの連携が容易になる

## イシュー作成時のルール

新しいイシューを作成する際は、以下を必ず設定してください：

1. **ラベル**: 「読み物」ラベルを付与
2. **プロジェクト**: 「読み物保守」プロジェクトに関連付け

```
# イシュー作成時の設定例
- ラベル: 読み物
- プロジェクト: 読み物保守
- 担当者: 必要に応じて設定
```

## 作業フロー

1. Linearでイシューを確認または作成
2. イシューIDでブランチを作成（`git checkout -b EDG-XXX`）
3. 作業を実施
4. コミット・プッシュ
5. PRを作成（Linearイシューへのリンクを含める）
6. レビュー・マージ後、Linearのステータスを更新
